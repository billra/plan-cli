Plan.py Test Suite
==================

Initial State
-------------
The file should start empty.
    >>> plan_file.exists()
    False

Adding Tasks & Formatting
-------------------------
We can add a hierarchy in a single atomic command. We also test that quote
and backslash escaping works cleanly.
    >>> cmd("1", "Root", "1.1", "Child A", "1.2", "Child B", "1.2.1", "Grandchild B.1", "1.3", "Child C")
    >>> cmd("1.4", "Escaped \"quotes\" and \\slash")

Reading Tasks
-------------
Running the script with no arguments should print the whole file.
    >>> cmd()
    ☐ 1 "Root"
    ☐ 1.1 "Child A"
    ☐ 1.2 "Child B"
    ☐ 1.2.1 "Grandchild B.1"
    ☐ 1.3 "Child C"
    ☐ 1.4 "Escaped \"quotes\" and \\slash"

Printing a specific subtree:
    >>> cmd("1.2")
    ☐ 1.2 "Child B"
    ☐ 1.2.1 "Grandchild B.1"

Validation Errors (Adding)
--------------------------
Let's test boundary conditions for numbering logic (gaps, missing parents, invalid syntax).
    >>> cmd("1.6", "Gap at the end")
    ValidationError: sibling gap - expected 1.5

    >>> cmd("3", "Root sibling gap")
    ValidationError: sibling gap - expected 2

    >>> cmd("1.2.3.1", "Missing parent")
    ValidationError: parent 1.2.3 does not exist

    >>> cmd("01.1", "Leading zero")
    ValidationError: invalid task number format: 01.1

    >>> cmd("1.0", "Zero segment")
    ValidationError: invalid task number format: 1.0

Completeness Propagation (Bubbling & Cascading)
-----------------------------------------------
Completing a child does not complete the parent if other siblings are incomplete.
However, since 1.2.1 is the ONLY child of 1.2, completing it fulfills the condition
for 1.2 to be completed automatically.
    >>> cmd("complete", "1.2.1")
    >>> cmd()
    ☐ 1 "Root"
    ☐ 1.1 "Child A"
    ☒ 1.2 "Child B"
    ☒ 1.2.1 "Grandchild B.1"
    ☐ 1.3 "Child C"
    ☐ 1.4 "Escaped \"quotes\" and \\slash"

Completing the root should cascade DOWN and complete everything.
    >>> cmd("complete", "1")
    >>> cmd()
    ☒ 1 "Root"
    ☒ 1.1 "Child A"
    ☒ 1.2 "Child B"
    ☒ 1.2.1 "Grandchild B.1"
    ☒ 1.3 "Child C"
    ☒ 1.4 "Escaped \"quotes\" and \\slash"

Replacing Tasks (Atomic State Reset)
------------------------------------
If we update an existing task, its description is updated, it resets to incomplete (☐),
and this incomplete state immediately bubbles UP to all ancestors.
    >>> cmd("1.2.1", "Updated Grandchild")
    >>> cmd()
    ☐ 1 "Root"
    ☒ 1.1 "Child A"
    ☐ 1.2 "Child B"
    ☐ 1.2.1 "Updated Grandchild"
    ☒ 1.3 "Child C"
    ☒ 1.4 "Escaped \"quotes\" and \\slash"

Incompleting a parent should cascade DOWN to children (and bubble UP).
    >>> cmd("incomplete", "1.3")
    >>> cmd()
    ☐ 1 "Root"
    ☒ 1.1 "Child A"
    ☐ 1.2 "Child B"
    ☐ 1.2.1 "Updated Grandchild"
    ☐ 1.3 "Child C"
    ☒ 1.4 "Escaped \"quotes\" and \\slash"

Insert Operations (Beginning, Middle, End, Errors)
--------------------------------------------------
Inserting at the BEGINNING (1.1). Shifts all current siblings up.
    >>> cmd("insert", "1.1", "New Child A")

Inserting in the MIDDLE (1.3). Shifts 1.3 and subsequent siblings up.
    >>> cmd("insert", "1.3", "New Child B")

Inserting at the END (1.7). Next logical sibling.
    >>> cmd("insert", "1.7", "End Child")

Check the tree after inserts:
    >>> cmd()
    ☐ 1 "Root"
    ☐ 1.1 "New Child A"
    ☒ 1.2 "Child A"
    ☐ 1.3 "New Child B"
    ☐ 1.4 "Child B"
    ☐ 1.4.1 "Updated Grandchild"
    ☐ 1.5 "Child C"
    ☒ 1.6 "Escaped \"quotes\" and \\slash"
    ☐ 1.7 "End Child"

Insert Error Check (Gap):
    >>> cmd("insert", "1.9", "Gap insert")
    ValidationError: sibling gap - expected 1.8

Delete Operations (Beginning, Middle, End, Subtrees)
----------------------------------------------------
Deleting at the END.
    >>> cmd("delete", "1.7")

Deleting at the BEGINNING. Shifts all remaining siblings down.
    >>> cmd("delete", "1.1")

Deleting in the MIDDLE (which also tests deleting a subtree).
Deleting 1.3 (which used to be 1.4 "Child B" and its child) should remove both
and close the gap.
    >>> cmd("delete", "1.3")

Verify final tree after all deletions:
    >>> cmd()
    ☐ 1 "Root"
    ☒ 1.1 "Child A"
    ☐ 1.2 "New Child B"
    ☐ 1.3 "Child C"
    ☒ 1.4 "Escaped \"quotes\" and \\slash"

Delete Error Check:
    >>> cmd("delete", "1.5")
    ValidationError: task 1.5 does not exist
