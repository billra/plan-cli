Plan.py Test Suite (Fixed ID Architecture)
==========================================

Initial State & Lazy File Creation
----------------------------------
The file starts out non-existent. Simply displaying the plan should NOT create the file.
    >>> plan_file.exists()
    False
    >>> cmd()
    >>> plan_file.exists()
    False

Tasks can be added. Under the fixed ID model, we don't strictly require starting at 1.
Writing a task triggers the actual file creation.
    >>> cmd("1", "First Root")
    >>> plan_file.exists()
    True
    >>> cmd()
    ☐ 1 "First Root"

Atomic Addition & Order Independence
------------------------------------
Because `add_or_replace` internally sorts tuples before processing, we should be able to
define a valid tree in a completely random order, and it will construct properly without
throwing orphan errors.
    >>> cmd("2.1.1", "Leaf", "2", "Root", "2.1", "Branch")
    >>> cmd("2")
    ☐ 2 "Root"
    ☐ 2.1 "Branch"
    ☐ 2.1.1 "Leaf"

Orphan Validation
-----------------
Adding a task missing its immediate parent fails validation.
    >>> cmd("3.1", "Orphan")
    PlanError: ...

    >>> cmd("2.2.1", "Deep Orphan")
    PlanError: ...

Gaps Are Completely Legal
-------------------------
Unlike the old model, jumping from 1 to 5, or 2.1 to 2.9, is perfectly fine.
    >>> cmd("5", "Far Root", "2.9", "Far Branch")
    >>> cmd()
    ☐ 1 "First Root"
    ☐ 2 "Root"
    ☐ 2.1 "Branch"
    ☐ 2.1.1 "Leaf"
    ☐ 2.9 "Far Branch"
    ☐ 5 "Far Root"

If a batch contains an invalid operation (like an orphan), the entire command aborts atomically.
    >>> cmd("2.2", "Valid", "4.1", "Invalid Orphan")
    PlanError: ...

Upserts & Overwrites
--------------------
Replacing an existing task updates its description AND forces its state to incomplete.
Because state is decoupled, this does NOT bubble up or down.
    >>> cmd("complete", "2.1")
    >>> cmd("2.1", "Updated Branch")
    >>> cmd("2")
    ☐ 2 "Root"
    ☐ 2.1 "Updated Branch"
    ☐ 2.1.1 "Leaf"
    ☐ 2.9 "Far Branch"

Decoupled State (No Automagic)
------------------------------
Tasks are independent. Completing a parent does NOT complete its children.
    >>> cmd("complete", "2")
    >>> cmd("2")
    ☒ 2 "Root"
    ☐ 2.1 "Updated Branch"
    ☐ 2.1.1 "Leaf"
    ☐ 2.9 "Far Branch"

Completing a child does NOT dynamically complete the parent.
    >>> cmd("complete", "2.1.1")
    >>> cmd("2")
    ☒ 2 "Root"
    ☐ 2.1 "Updated Branch"
    ☒ 2.1.1 "Leaf"
    ☐ 2.9 "Far Branch"

Structural Deletion (Subtree Wiping)
------------------------------------
Deleting a node wipes out that node and all of its descendants, to prevent orphans.
Deleting 2.1 should wipe 2.1 and 2.1.1, leaving 2 and 2.9 alone. It does not shift 2.9!
    >>> cmd("delete", "2.1")
    >>> cmd("2")
    ☒ 2 "Root"
    ☐ 2.9 "Far Branch"

Deleting the root 2 wipes the rest of that tree.
    >>> cmd("delete", "2")
    >>> cmd()
    ☐ 1 "First Root"
    ☐ 5 "Far Root"

Attempting to add a child to a deleted parent fails.
    >>> cmd("2.1", "Zombie Child")
    PlanError: ...

String Escaping & Whitespace Preservation
-----------------------------------------
Double quotes and backslashes must be safely escaped in storage and unescaped in memory.
Tasks can also have completely empty descriptions or descriptions with leading/trailing spaces.
    >>> cmd("6", 'Has "quotes" and \\slashes', "6.1", "", "6.2", "  Spaces  ")
    >>> cmd("6")
    ☐ 6 "Has \"quotes\" and \\slashes"
    ☐ 6.1 ""
    ☐ 6.2 "  Spaces  "

Target Missing Errors
---------------------
Attempting to operate on non-existent targets safely throws validation errors.
    >>> cmd("complete", "9.9")
    PlanError: ...

    >>> cmd("delete", "9.9")
    PlanError: ...

Invalid Number Parsing (Strict Regex)
-------------------------------------
The `parse_num` regex requires strict dot-separated integers.
No leading or trailing dots:
    >>> cmd("delete", "1.")
    PlanError: ...
    >>> cmd("delete", ".1")
    PlanError: ...

No letters or symbols:
    >>> cmd("delete", "A.1")
    PlanError: ...

No zeros or leading zeros:
    >>> cmd("delete", "0")
    PlanError: ...
    >>> cmd("delete", "1.0")
    PlanError: ...
    >>> cmd("delete", "01.2")
    PlanError: ...

Multi-Digit & Deep Nesting Numbering
------------------------------------
Tuples must sort mathematically, not lexically (e.g., 20.300 comes after 20.9).
    >>> cmd("20", "Twenty", "20.9", "Nine", "20.300", "Three Hundred")
    >>> cmd("20")
    ☐ 20 "Twenty"
    ☐ 20.9 "Nine"
    ☐ 20.300 "Three Hundred"

Targeted Display Filtering
--------------------------
Ensure targeting a sub-branch strictly prints ONLY that branch and nothing above or beside it.
    >>> cmd("20.9")
    ☐ 20.9 "Nine"

CLI Dispatch: Argument Count Mismatches
---------------------------------------
Batch upsert requires pairs. An odd number of arguments triggers the fallback error.
    >>> cmd("8", "Valid", "9")
    PlanError: ...

Commands requiring exactly one argument fail safely if given zero or multiple.
    >>> cmd("complete")
    PlanError: ...
    >>> cmd("delete", "1", "2")
    PlanError: ...

CLI Dispatch: Unrecognized Commands
-----------------------------------
Arbitrary strings are caught and rejected gracefully.
    >>> cmd("magic", "1")
    PlanError: ...

Help Flag Safety
----------------
Running help should print the text but should NOT wipe or overwrite the plan file.
    >>> cmd("--help")
    plan - A Hierarchical Task Manager...

File Parsing Resilience (Empty Lines & Corruption)
--------------------------------------------------
The tool safely ignores empty lines in the storage file.
    >>> _ = plan_file.write_text("\n\n☐ 1 \"Surviving Task\"\n\n\n", encoding="utf-8")
    >>> cmd()
    ☐ 1 "Surviving Task"

If a user manually corrupts the file with malformed text, the tool aborts to protect data.
    >>> _ = plan_file.write_text("☐ 1 Missing Quotes Around Description\n", encoding="utf-8")
    >>> cmd()
    OSError: ...

File Load Validation (Orphans)
------------------------------
If a user manually created an orphan in the text file, the loader must catch it and abort.
Notice 1.2.1 is missing its required parent 1.2.
    >>> _ = plan_file.write_text("☐ 1 \"Root\"\n☐ 1.2.1 \"Orphan\"\n", encoding="utf-8")
    >>> cmd()
    OSError: ...
