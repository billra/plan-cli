Plan.py Test Suite (Explicit Command Architecture)
================================================

Initial State & Empty Plan Fallback
-----------------------------------
The file starts out non-existent. When the plan is empty, running without arguments
falls back to showing the help text instead of an empty screen.
    >>> plan_file.exists()
    False
    >>> cmd()
    plan - A Hierarchical Task Manager...
    >>> plan_file.exists()
    False

Tasks can be added using the explicit 'add' command.
Writing a task triggers the actual file creation.
    >>> cmd("add", "1", "First Root")
    >>> plan_file.exists()
    True
    >>> cmd("ls")
    ☐ 1 First Root

Auto-joining Descriptions & Whitespace
--------------------------------------
The 'add' and 'edit' commands automatically join multiple text arguments with a space.
Leading and trailing spaces are silently stripped. Empty descriptions are rejected.
    >>> cmd("add", "2", "Root", "Task", "with", "multiple", "words")
    >>> cmd("add", "2.1", "  Spaces Stripped  ")
    >>> cmd("ls", "2")
    ☐ 2 Root Task with multiple words
    ☐ 2.1 Spaces Stripped

    >>> cmd("add", "2.2")
    PlanError: ...
    >>> cmd("add", "2.2", "   ")
    PlanError: ...

Duplicate ID & Orphan Validation
--------------------------------
Adding an ID that already exists fails.
    >>> cmd("add", "1", "Duplicate")
    PlanError: ...

Adding a task missing its immediate parent fails validation.
    >>> cmd("add", "3.1", "Orphan")
    PlanError: ...
    >>> cmd("add", "2.3.1", "Deep Orphan")
    PlanError: ...

Sequential Addition & Gaps
--------------------------
Gaps are completely legal (e.g., jumping from 1 to 5).
    >>> cmd("add", "2.1.1", "Leaf")
    >>> cmd("add", "5", "Far Root")
    >>> cmd("add", "2.9", "Far Branch")
    >>> cmd("ls")
    ☐ 1 First Root
    ☐ 2 Root Task with multiple words
    ☐ 2.1 Spaces Stripped
    ☐ 2.1.1 Leaf
    ☐ 2.9 Far Branch
    ☐ 5 Far Root

Editing Tasks (State Preservation)
----------------------------------
Editing an existing task updates its description. Unlike the old 'upsert' model,
'edit' requires the ID to exist and preserves the current done/todo state.
    >>> cmd("done", "2.1")
    >>> cmd("edit", "2.1", "Updated", "Branch")
    >>> cmd("ls", "2.1")
    ☒ 2.1 Updated Branch
    ☐ 2.1.1 Leaf

    >>> cmd("edit", "9.9", "Ghost Task")
    PlanError: ...

Decoupled State (No Automagic)
------------------------------
Tasks are independent. Completing a parent does NOT complete its children.
    >>> cmd("done", "2")
    >>> cmd("ls", "2")
    ☒ 2 Root Task with multiple words
    ☒ 2.1 Updated Branch
    ☐ 2.1.1 Leaf
    ☐ 2.9 Far Branch

Completing a child does NOT dynamically complete the parent.
    >>> cmd("done", "2.1.1")
    >>> cmd("todo", "2.1")
    >>> cmd("ls", "2")
    ☒ 2 Root Task with multiple words
    ☐ 2.1 Updated Branch
    ☒ 2.1.1 Leaf
    ☐ 2.9 Far Branch

Structural Deletion (Subtree Wiping)
------------------------------------
Deleting a node wipes out that node and all of its descendants, to prevent orphans.
Deleting 2.1 should wipe 2.1 and 2.1.1, leaving 2 and 2.9 alone.
    >>> cmd("rm", "2.1")
    >>> cmd("ls", "2")
    ☒ 2 Root Task with multiple words
    ☐ 2.9 Far Branch

Deleting the root 2 wipes the rest of that tree.
    >>> cmd("rm", "2")
    >>> cmd("ls")
    ☐ 1 First Root
    ☐ 5 Far Root

Target Missing Errors
---------------------
Attempting to operate on non-existent targets safely throws validation errors.
    >>> cmd("done", "9.9")
    PlanError: ...
    >>> cmd("todo", "9.9")
    PlanError: ...
    >>> cmd("rm", "9.9")
    PlanError: ...
    >>> cmd("ls", "9.9")
    PlanError: ...

Invalid Number Parsing (Strict Regex)
-------------------------------------
The parser requires strict dot-separated integers. No leading/trailing dots or symbols.
    >>> cmd("rm", "1.")
    PlanError: ...
    >>> cmd("rm", ".1")
    PlanError: ...
    >>> cmd("rm", "A.1")
    PlanError: ...
    >>> cmd("rm", "0")
    PlanError: ...
    >>> cmd("rm", "1.0")
    PlanError: ...

Multi-Digit & Deep Nesting Numbering
------------------------------------
Tuples must sort mathematically, not lexically (e.g., 20.300 comes after 20.9).
    >>> cmd("add", "20", "Twenty")
    >>> cmd("add", "20.9", "Nine")
    >>> cmd("add", "20.300", "Three Hundred")
    >>> cmd("ls", "20")
    ☐ 20 Twenty
    ☐ 20.9 Nine
    ☐ 20.300 Three Hundred

CLI Dispatch: Argument Validation
---------------------------------
Commands requiring exactly one argument fail safely if given zero or multiple.
    >>> cmd("rm")
    PlanError: ...
    >>> cmd("rm", "1", "2")
    PlanError: ...
    >>> cmd("done")
    PlanError: ...
    >>> cmd("todo", "1", "extra")
    PlanError: ...

Commands requiring ID + text fail if text is omitted.
    >>> cmd("edit", "1")
    PlanError: ...

Arbitrary strings are caught and rejected gracefully.
    >>> cmd("magic", "1")
    PlanError: ...

Help Flag Safety
----------------
Running help should print the text but should NOT wipe or overwrite the plan file.
    >>> cmd("--help")
    plan - A Hierarchical Task Manager...

Deep Hierarchy Lifecycle & ID Reincarnation
-------------------------------------------
Testing a long chain of events on a deep hierarchy to ensure state doesn't leak.
    >>> cmd("add", "10", "Deep Root")
    >>> cmd("add", "10.1", "Child")
    >>> cmd("add", "10.1.1", "Grandchild")
    >>> cmd("done", "10.1.1")
    >>> cmd("rm", "10.1")

After removing the middle child, the root should remain, but the grandchild is wiped.
    >>> cmd("ls", "10")
    ☐ 10 Deep Root

Re-adding a previously deleted ID should work perfectly and default to incomplete (☐).
    >>> cmd("add", "10.1", "Reincarnated Child")
    >>> cmd("ls", "10")
    ☐ 10 Deep Root
    ☐ 10.1 Reincarnated Child

Unicode & Special Characters
----------------------------
Because files are forced to UTF-8, emojis and non-Latin characters parse safely.
    >>> cmd("add", "11", "Buy groceries 🍎, 🍞, 🍷")
    >>> cmd("add", "11.1", "日本語のテスト (Japanese text test)")
    >>> cmd("ls", "11")
    ☐ 11 Buy groceries 🍎, 🍞, 🍷
    ☐ 11.1 日本語のテスト (Japanese text test)

Duplicate Descriptions
----------------------
Task descriptions do not need to be unique. Only the IDs act as primary keys.
    >>> cmd("add", "12", "Check emails")
    >>> cmd("add", "13", "Check emails")
    >>> cmd("ls", "12")
    ☐ 12 Check emails
    >>> cmd("ls", "13")
    ☐ 13 Check emails

Command Case-Sensitivity & Malformed IDs
----------------------------------------
Commands must be strictly lowercase.
    >>> cmd("ADD", "14", "Test")
    PlanError: ...
    >>> cmd("Done", "12")
    PlanError: ...

ID parsing naturally tolerates spaces but rejects double-dots.
    >>> cmd("add", "15", "Parent for padded ID")
    >>> cmd("add", " 15.1 ", "Padded ID")
    >>> cmd("ls", "15.1")
    ☐ 15.1 Padded ID
    >>> cmd("add", "15..2", "Double dot")
    PlanError: ...

Editing the Root
----------------
Editing a parent node must not accidentally orphan or sever its children.
    >>> cmd("add", "16", "Old Root")
    >>> cmd("add", "16.1", "Safe Child")
    >>> cmd("edit", "16", "New Root")
    >>> cmd("ls", "16")
    ☐ 16 New Root
    ☐ 16.1 Safe Child

Out-of-Order Addition & Strict Sorting
--------------------------------------
Let's wipe the file clean to test a fresh scenario.
    >>> _ = plan_file.write_text("", encoding="utf-8")

Because the plan is stored in a dictionary, adding tasks in a completely
random order must still result in a perfectly sorted output.
    >>> cmd("add", "30", "Thirty")
    >>> cmd("add", "10", "Ten")
    >>> cmd("add", "20", "Twenty")
    >>> cmd("add", "10.1", "Ten point one")
    >>> cmd("ls")
    ☐ 10 Ten
    ☐ 10.1 Ten point one
    ☐ 20 Twenty
    ☐ 30 Thirty

Total Depletion Fallback
------------------------
If all tasks are deleted from the file, the application should gracefully return
to the initial "empty plan" state and show the help text on a blank run.
    >>> cmd("rm", "10")
    >>> cmd("rm", "20")
    >>> cmd("rm", "30")
    >>> cmd("ls")
    >>> cmd()
    plan - A Hierarchical Task Manager...

File Parsing Resilience (Empty Lines & Corruption)
--------------------------------------------------
!!! WARNING: Destructive file edits begin here !!!
The tool safely ignores empty lines in the storage file.
    >>> _ = plan_file.write_text("\n\n☐ 1 Surviving Task\n\n\n", encoding="utf-8")
    >>> cmd("ls")
    ☐ 1 Surviving Task

If a user manually corrupts the file with malformed text, the tool aborts to protect data.
    >>> _ = plan_file.write_text("? 1 Invalid State Character\n", encoding="utf-8")
    >>> cmd("ls")
    OSError: ...

If a file contains a record where the description parses to empty, it aborts.
    >>> _ = plan_file.write_text("☐ 1    \n", encoding="utf-8")
    >>> cmd("ls")
    OSError: ...

File Load Validation (Orphans)
------------------------------
If a user manually creates an orphan in the text file, the loader must catch it and abort.
Notice 1.2.1 is missing its required parent 1.2.
    >>> _ = plan_file.write_text("☐ 1 Root\n☐ 1.2.1 Orphan\n", encoding="utf-8")
    >>> cmd("ls")
    OSError: ...
